## Component Inventory

### Onboarding Flow (Figma `210:58803`) [FIGMA-PENDING]

The onboarding flow is a multi-step modal or full-page overlay that appears on first login. Based on ONBD-01 and ONBD-02:

| Component | Source | Notes |
|-----------|--------|-------|
| Dialog (overlay) | shadcn `dialog` | Full-screen or centered modal container |
| Button | shadcn `button` | "Next", "Skip", "Get Started" actions |
| Progress indicator | Custom | Step dots or progress bar showing current position |
| Illustration container | Custom div | Holds step-specific illustration/graphic |

**Interaction flow:**

1. **Trigger:** Onboarding appears automatically after first successful login. Detection: check if user has zero completed audits AND has never dismissed onboarding (stored in localStorage key `auditflow_onboarding_completed`).
2. **Step 1 — Welcome:** Greeting + brief value proposition ("Run your first accessibility audit in 60 seconds").
3. **Step 2 — Enter URL:** Explains the scan input — "Paste any website URL and select how many pages to crawl."
4. **Step 3 — Review Results:** Explains the results page — "See your WCAG score, violations by severity, and AI-generated fix instructions."
5. **Step 4 — Download Report:** Explains PDF export — "Download a professional PDF report to share with your team."
6. **Navigation:** "Next" button advances, "Skip" link dismisses entire flow. Both set `localStorage.setItem('auditflow_onboarding_completed', 'true')`.
7. **Completion:** Final step has "Get Started" button that navigates to `/scan`.

### Tutorial Flow (Figma `210:58819`) [FIGMA-PENDING]

Tutorial is a contextual tooltip/spotlight overlay that guides users through the dashboard UI after onboarding completes (or on demand from a help menu).

| Component | Source | Notes |
|-----------|--------|-------|
| Popover | shadcn `popover` | Tooltip-like overlay anchored to UI elements |
| Button | shadcn `button` | "Next", "Done" navigation within tutorial |
| Overlay backdrop | Custom div | Semi-transparent backdrop with spotlight cutout |

**Interaction flow:**

1. **Trigger:** Automatically after onboarding "Get Started" OR accessible via a "?" help button in the dashboard header. Detection: `localStorage` key `auditflow_tutorial_completed`.
2. **Step 1:** Spotlight the "+ New Scan" button in the sidebar or header.
3. **Step 2:** Spotlight the recent audits list on the dashboard.
4. **Step 3:** Spotlight the credit balance indicator.
5. **Navigation:** "Next" advances spotlight, "Skip" dismisses. "Done" on final step.
6. **Completion:** Sets `localStorage.setItem('auditflow_tutorial_completed', 'true')`.

## State Management

| State | Storage | Key |
|-------|---------|-----|
| Onboarding completed | localStorage | `auditflow_onboarding_completed` |
| Tutorial completed | localStorage | `auditflow_tutorial_completed` |
| Current onboarding step | React useState | Component-local |
| Current tutorial step | React useState | Component-local |

**Why localStorage, not database:** Onboarding is a low-stakes UX convenience. If a user clears storage and sees onboarding again, that is acceptable. Avoiding an API call keeps the flow instant and dependency-free.

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Focus trap | Dialog component provides focus trap automatically (Radix Dialog) |
| Keyboard nav | Tab through Skip/Next buttons, Enter to activate, Escape to skip |
| Screen reader | aria-label on step indicators, role="progressbar" or aria-valuenow on step dots |
| Tutorial spotlight | aria-describedby linking popover to target element |
| Reduced motion | Animations disabled per global CSS rule |
