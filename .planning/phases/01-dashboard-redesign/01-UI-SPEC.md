## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, table cell vertical padding |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, card padding on desktop |
| xl | 32px | Gap between stat cards row and audits table |
| 2xl | 48px | Page horizontal padding on large screens |
| 3xl | 64px | Not used in this phase |

Exceptions: none

## Typography

| Role | Size | Weight | Line Height | Tailwind Classes |
|------|------|--------|-------------|------------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm font-normal leading-normal` |
| Label | 12px | 500 (medium) | 1.33 | `text-xs font-medium` |
| Heading | 24px | 700 (bold) | 1.2 | `text-2xl font-bold` |
| Stat Value | 28px | 700 (bold) | 1.2 | `text-[28px] font-bold` |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` / `#FFFFFF` | Page background, card backgrounds |
| Secondary (30%) | `hsl(210 40% 96.1%)` / `#F1F5F9` | Table header background, search input background, skeleton placeholder fill |
| Accent (10%) | `hsl(239 84% 67%)` / `#4F46E5` | Primary CTA buttons, active nav links, URL link text, icon highlights |
| Destructive | `hsl(0 84.2% 60.2%)` / `#EF4444` | Delete button hover state only |

Accent reserved for:
- "New Scan" primary button
- "Run Your First Scan" empty state CTA
- URL text links in audit table rows
- Stat card icon backgrounds (indigo-50 tint)
- Search input focus ring

## Component Inventory

### DASH-01: Stat Cards (4 cards in a row)

| Card | Label | Value Source | Icon | Icon BG |
|------|-------|-------------|------|---------|
| Total Audits | `audits.length` | `FileText` | `bg-blue-50 text-blue-600` |
| Avg WCAG Score | `Math.round(avg of completed wcag_score)` + "%" | `BarChart2` | `bg-orange-50 text-orange-500` |
| Critical Issues | `sum of critical_count across all audits` | `AlertTriangle` | `bg-red-50 text-red-600` |
| Compliant Sites | `count of audits where wcag_score >= 80` | `ShieldCheck` | `bg-green-50 text-green-600` |

Layout: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4`
Card structure: `rounded-xl border border-border bg-card p-5 shadow-sm`
Icon container: `h-8 w-8 rounded-lg flex items-center justify-center`
Value: `text-[28px] font-bold text-foreground`
Label: `text-xs font-medium text-muted-foreground mb-2`

### DASH-02: Loading Skeletons

Skeleton component: use shadcn `<Skeleton />` from `src/components/ui/skeleton`
- 4 stat card skeletons: same card dimensions, internal `<Skeleton className="h-4 w-20 mb-2" />` for label, `<Skeleton className="h-8 w-16" />` for value
- Table skeleton: 5 rows, each row has 6 `<Skeleton />` cells matching column widths
- Shimmer animation: built into shadcn Skeleton (pulse animation)
- Show skeletons when `loading === true` from `useAudits()` hook

### DASH-03: Empty State

Container: centered within the table card area, `py-20 px-6 text-center`
Icon: `ScanSearch` in a `h-16 w-16 rounded-2xl bg-indigo-50` container
Heading: "No audits yet" -- `text-base font-semibold text-foreground`
Body: "Run your first accessibility scan to see results here. It only takes a minute." -- `text-sm text-muted-foreground max-w-xs mx-auto`
CTA: `<Button>` with `rounded-full bg-primary text-primary-foreground px-6` -- label: "Run Your First Scan"
CTA navigates to `/scan`

### DASH-04: Recent Audits Table

Table container: `rounded-xl border border-border bg-card shadow-sm overflow-hidden`
Toolbar: search input (left) + "New Scan" button (right), separated by `border-b border-border`
Search: `<input>` with `Search` icon, `w-56 pl-9 pr-3 py-1.5 text-sm rounded-lg bg-secondary`

Table columns:

| Column | Width | Content |
|--------|-------|---------|
| URL | max-w-[220px] | Lucide link icon + truncated URL, `text-primary` |
| Date | auto | `DD/MM/YYYY` format via `toLocaleDateString('en-GB')` |
| WCAG Score | auto | Score number with color-coded text per score thresholds |
| Status | auto | `<StatusBadge>` pill component |
| Pages | auto | `pages_scanned` number |
| Actions | auto | Icon buttons: View (Eye), Download (Download), Delete (Trash2) |

Row: `hover:bg-muted/50 cursor-pointer transition-colors`, clickable to `/audits/:id`
Footer: `Total {n} items` -- `text-xs text-muted-foreground px-4 py-3 border-t border-border`

### PLSH-01: Skeleton Pattern (reusable)

Create a `DashboardSkeleton` component that renders:
1. Page heading skeleton: `<Skeleton className="h-7 w-48 mb-1" />` + `<Skeleton className="h-4 w-72 mb-6" />`
2. 4 stat card skeletons in grid
3. Table card with toolbar skeleton + 5 row skeletons

This pattern will be replicated for Reports and AuditDetail in Phase 2.

## Interaction States

| Element | State | Behavior |
|---------|-------|----------|
| Stat cards | Loading | Skeleton placeholders (pulse animation) |
| Stat cards | Loaded | Populated with computed values from API data |
| Stat cards | Zero audits | Show "0" for Total Audits, "--" for Avg Score, "0" for Critical, "0" for Compliant |
| Audit table | Loading | 5-row skeleton |
| Audit table | Empty (0 audits) | Empty state with icon, copy, and CTA |
| Audit table | Populated | Rows with hover highlight, clickable |
| Table row | Hover | `bg-muted/50` background transition |
| Search input | Focus | `ring-1 ring-primary border-primary` |
| Delete button | Hover | `text-red-600 bg-red-50` |
| Delete button | Click | Browser `confirm()` dialog (existing pattern) |
| "New Scan" button | Click | Navigate to `/scan` |
| Status badge (scanning) | Idle | Pulsing dot animation via `animate-pulse` |
